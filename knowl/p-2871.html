The next step is to look at the vertices neighboring \(v\) (see Figure 9.20.4). This step results in no changes to the graph, so we move on to node \(y\text{.}\) At node \(y\) (see Figure 9.20.5) we discover that it is cheaper to get to both \(w\) and \(z\text{,}\) so we adjust the distances and predecessor links accordingly. Finally we check nodes \(w\) and \(z\) (see see Figure 9.20.6 and see Figure 9.20.7). However, no additional changes are found and so the priority queue is empty and Dijkstra's algorithm exits.
