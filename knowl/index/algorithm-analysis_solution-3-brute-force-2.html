A brute force technique for solving a problem typically tries to exhaust all possibilities. For the anagram detection problem, we can simply generate an array of all possible strings using the characters from s1 and then see if s2 occurs. However, there is a difficulty with this approach. When generating all possible strings from s1, there are n possible first characters, \(n-1\) possible characters for the second position, \(n-2\) for the third, and so on. The total number of candidate strings is \(n*(n-1)*(n-2)*...*3*2*1\text{,}\) which is \(n!\text{.}\) Although some of the strings may be duplicates, the program cannot know this ahead of time and so it will still generate \(n!\) different strings.
