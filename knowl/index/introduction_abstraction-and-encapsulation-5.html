The object-oriented principle of encapsulation refers to the practice of grouping or encapsulating related data and the methods that operate on that data into a single unit, typically a class. Encapsulation also requires we hide most of the internal contents of that class, except what is absolutely necessary to expose. Hence, we will restrict the access to our class as much as we can, so that a user can change the class properties and behaviors only from methods provided by the class. C++ allows us to control access with the access keywords public and private. It is typical in C++ to make all data attributes private and most methods public. All attribute variables under the private keyword will only be able to be accessed by the object’s class methods, not by the user. Only C++’s public methods can be accessed and used by the user. Because we want our user to be able to call every constructor directly, we always place the constructor under public. A third access keyword, protected will be discussed later.
