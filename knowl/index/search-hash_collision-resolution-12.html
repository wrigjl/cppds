The general name for this process of looking for another slot after a collision is rehashing. With simple linear probing, the rehash function is \(newhashvalue = rehash(oldhashvalue)\) where \(rehash(pos) = (pos + 1) \% sizeoftable\text{.}\) The “plus 3” rehash can be defined as \(rehash(pos) = (pos+3) \% sizeoftable\text{.}\) In general, \(rehash(pos) = (pos + skip) \% sizeoftable\text{.}\) It is important to note that the size of the “skip” must be such that all the slots in the table will eventually be visited. Otherwise, part of the table will be unused. To ensure this, it is often suggested that the table size be a prime number. This is the reason we have been using 11 in our examples.
