As we saw with unordered linked lists, it is necessary to have an additional reference, again called previous, since current will not provide access to the node that must be modified. Listing 4.6.7 shows the complete add method. Lines 2–3 set up the two external references and lines 9–10 again allow previous to follow one node behind current every time through the iteration. The condition (line 5) allows the iteration to continue as long as there are more nodes and the value in the current node is not larger than the item. In either case, when the iteration fails, we have found the location for the new node.
